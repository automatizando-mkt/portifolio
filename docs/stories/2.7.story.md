## Story

**As a** potencial cliente,
**I want** uma página de contato clara com formulário visível,
**so that** eu possa entrar em contato facilmente.

2. Validação client-side dos campos obrigatórios com feedback visual
3. Informações de contato alternativas (email, redes sociais) ao lado do formulário
4. Layout responsivo (form empilhado em mobile, side-by-side em desktop)
5. Formulário ainda não envia dados (funcionalidade no Epic 4) — exibe mensagem de placeholder no submit

## Dev Notes

### Contact Form — Data Model

```typescript
// Referência para Epic 4 (implementação real do envio)
import { z } from 'zod';

export const contactSchema = z.object({
  name: z.string().min(2, 'Nome é obrigatório'),
  email: z.string().email('Email inválido'),
  company: z.string().optional(),
  message: z.string().min(10, 'Mensagem deve ter pelo menos 10 caracteres'),
});

export type ContactFormData = z.infer<typeof contactSchema>;
```
[Source: docs/architecture/data-models.md]

**Nota:** Nesta story usamos validação HTML nativa. A validação com Zod + react-hook-form será implementada no Epic 4 quando o formulário for conectado a um serviço real.

### Canvas DS — Input Styling

Não há pattern de input no Canvas DS original. Adaptar o dark theme:
- Background: transparente (bg-transparent)
- Border: `border-white/10` → `focus:border-canvas-accent`
- Text: `text-white`, placeholder `text-neutral-600`
- Sem border-radius (manter angular, coerente com buttons)
- Transição: `transition-colors duration-300`
